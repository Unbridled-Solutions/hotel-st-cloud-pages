AI Chief of Staff — Infrastructure Architecture

End-to-end system: customer onboarding → agent runtime → integrations → backup & recovery

What the Customer Experiences

  • • Signs up via web portal, completes onboarding questionnaire
  • • Gets paired with their agent via Telegram in minutes
  • • Talks to it like a person — text or voice
  • • Agent remembers everything, works autonomously overnight
  • • Monthly check-in expands capabilities over time

Infrastructure Per Customer

  • • 1× Hetzner cloud server (CCX23 or similar)
  • • Hermes Agent runtime + configured identity
  • • Backblaze B2 bucket (nightly + weekly backups)
  • • Uptime monitor with Telegram alerting
  • • Scoped API keys per integration
  • • Obsidian vault — knowledge base + runbook

Security Principles

  • • Zero-retention models for legal / sensitive data
  • • Credentials never stored in chat or plain memory
  • • Scoped API keys — least privilege per service
  • • Restricted sudoers — script-level only, no shell
  • • Encrypted backups (restic + customer passphrase)
  • • Private B2 bucket — no public access

Vertical Skill Packs

  • • 🏨 Hospitality — PMS, booking, guest comms
  • • ⚖️ Legal — drafting, review, CO-aware routing
  • • 📈 Advertising — Google Ads, landing pages, reports
  • • 🏠 Real Estate — listings, CRM, comps
  • • 🏥 Medical / Financial — compliance-routed
  • • ⚙️ Custom — built during onboarding

Disaster Recovery

  • • Hermes tarball backed up nightly to B2
  • • Full server snapshot weekly (encrypted restic)
  • • 3-month snapshot history retained
  • • RTO < 1 hour on new server from scratch
  • • Runbook stored in Obsidian vault
  • • Gateway uptime monitor — alert within 5 min

Business Model

  • • Done-for-you: ~$750–1,500/month per customer
  • • Setup fee covers onboarding + identity build
  • • Monthly retainer: server + backups + maintenance
  • • Upsell: additional integrations, larger agent teams
  • • White-label: license to agencies (royalty/seat)
  • • Proof of concept: Hotel St. Cloud ✅
