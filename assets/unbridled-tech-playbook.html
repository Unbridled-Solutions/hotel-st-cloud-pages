Universal Typography Rules

/* ✅ Always use clamp() — never bare px on headings or outer padding */
h1 { font-size: clamp(2rem, 5vw, 3.5rem); }
h2 { font-size: clamp(1.5rem, 3.5vw, 2.25rem); }
body { font-size: 16px; /* min 16px — never below 14px anywhere */ }

/* ✅ Container — always max-width, NEVER fixed width */
.container { max-width: 1100px; width: 100%; margin: 0 auto; }

/* ✅ Outer padding uses clamp() so it breathes on mobile */
.section { padding: clamp(2rem, 6vw, 4rem) clamp(1rem, 5vw, 3rem); }

Component Rules

✓ Do

Mobile-first always

Write base styles for mobile. Layer up with @media (min-width: 640px). Test mentally at 375px (iPhone SE).

✓ Do

Touch targets 44px min

Every button and tap target must be at least 44×44px. Users are on phones — fingertips, not cursors.

✓ Do

Grid collapses at 640px

Multi-column grids use @media (max-width: 640px) { grid-template-columns: 1fr; } — no orphan cards.

✕ Never

Fixed width containers

Never width: 1000px on any wrapper. Always max-width: Xpx; width: 100%. Fixed width = broken mobile = caught immediately.

✕ Never

External frameworks unlocked

No Bootstrap, Tailwind, or Vue without a pinned CDN version. Pin to exact semver, never @latest.

✕ Never

SVG elk logo custom-drawn

The elk is a covenant symbol. Always use the official PNG assets. Never recreate in SVG — it will look wrong.

02 · Architecture

Project Structure Patterns

Every project is a single self-contained HTML file. No build step. No bundler. No npm for the output artifact. All CSS and JS inline via <style> and <script>.

Repository Layout — hotel-st-cloud-pages

hotel-st-cloud-pages/ ├── assets/ # All Unbridled tools, dashboards, trackers │ ├── mc-overview.html # One-pager pattern reference │ ├── eagles-landing-tracker.html # Punch list tracker reference │ ├── eagles-landing-proforma.html # STR pro forma reference │ └── logos/ # Brand logo PNGs (no SVG) │ ├── unbridled-elk-icon-white.png │ └── unbridled-logo-white.png ├── fremontmakers/ # Fremont Makers brand subdirectory ├── standardoil/ # Standard Oil Coffee subdirectory ├── <brand-slug>/ # Each brand gets its own subdirectory ├── src/index.js # Cloudflare Worker — all API routes here └── wrangler.jsonc # Worker config (NOT Pages)

Naming Conventions

✓ Do

Lowercase, hyphens

All filenames: wine-club-tracker.html, labor-planner.html. No spaces, no caps, no underscores.

✓ Do

Descriptive slugs

mark-wine-app.html not tool1.html. Name by purpose — it's the URL the user shares.

✕ Never

Bare directories without index.html

If you deploy to /wine/ it must contain index.html. Cloudflare won't auto-redirect bare paths.

Multi-brand rule: Each brand gets a subdirectory — /fremontmakers/, /standardoil/, /<brand-slug>/. General Unbridled tools and dashboards live in /assets/. Never mix brand content into another brand's directory.

04 · Infrastructure

Deployment Practices

The Unbridled stack is Cloudflare Workers with Assets — not Cloudflare Pages. Git push alone does not deploy. Every deploy requires an explicit wrangler command.

Critical: This is a Cloudflare Worker project, not a Pages project. git push updates the repo only. You must run npm run deploy (or npx wrangler deploy) to publish changes to the live domain.

Deploy Workflow — Step by Step

1

Write the file

Write to /home/hermes/projects/hotel-st-cloud-pages/assets/. Use tee or file write tools — never python3 -c (string escaping breaks HTML).

2

Stage and commit

git add assets/your-file.html then git commit -m "feat: add tool name". Use conventional commits: feat: / fix: / chore:. Never use & in commit messages — shell treats it as background operator.

3

Push to GitHub

git push origin main. If auth error, source ~/.hermes/.env and inject GITHUB_TOKEN into the remote URL.

4

Deploy via Wrangler

npm run deploy from the project root — this runs wrangler deploy AND purges the Cloudflare edge cache. Do not skip this step.

5

Wait ~30 seconds, then verify

curl -s -o /dev/null -w '%{http_code}' -L https://offers.hotelstcloud.com/assets/your-file.html — must return 200.

6

Curl-verify the actual changed content

curl -sL 'https://offers.hotelstcloud.com/assets/your-file.html' | grep -o 'changed text here' | head -3 — Never say "it's deployed" based on wrangler output alone. Check the live content explicitly.

Versioning Rules

✓ Do

One file, one feature

Each tool is a self-contained HTML file. Version it by updating the same file. Don't create tool-v2.html — patch tool.html.

✓ Do

Redirect stubs for moved pages

If a page moves domains, leave a meta-refresh redirect stub at the original URL. Shared links stay working.

✕ Never

Read-file-then-write-back pattern

The read_file tool prepends line numbers. If you read a file and write it back via string ops, those line numbers get baked in. Use patch for surgical edits or write_file for full rewrites.

06 · Quality Gate

Pre-Deploy Checklist

Run this before every deploy. Nothing ships without passing these. They're not suggestions — they're the things that get caught immediately when missed.

✓
<meta name="viewport"> present in <head> — missing this = broken mobile, instant fail
✓
No fixed width: Xpx on outer container — use max-width + width: 100%
✓
Font sizes use clamp() — no bare px values on headings
✓
Multi-column grids collapse to 1fr at 640px
✓
Touch targets (buttons, links) are at least 44px tall
✓
All images are max-width: 100% — no stretching on narrow screens
✓
No placeholder text — no "Lorem ipsum", "TBD", "[INSERT]", "PLACEHOLDER"
✓
<title> and <meta name="description"> are populated with real copy
✓
Any CDN libraries are pinned to a specific version — not @latest
✓
Git commit message follows feat: / fix: / chore: convention — no & in message
✓
For JS-heavy files: run grep -n '\\`\|\\${' file.html — escaped backticks = silent JS failure
✓
curl returns 200 and confirms the changed text is live before telling anyone
✕
Never send HTML via Telegram file attachment as the deliverable — it can't be opened as a live page. Deploy first, share the URL.
✕
Never use auto-fit for a fixed-count grid (4 cards = use repeat(2, 1fr)) — orphan card layouts are caught immediately
The rule of thumb: Stan and Justin check deliverables on their phones, in Telegram's in-app browser. If it looks broken at 375px, it's wrong — full stop. Build mobile-first, verify mentally at that breakpoint, then deploy.
